# Product Requirements Document

## 1. Product / Feature Name

대형폐기물 품목 검색 서비스

## 2. Problem Statement

대형폐기물 배출 시 각 지자체에 등록된 행정상의 용어가 달라 키워드 기반 검색시 원하는 품목의 수수료 정보를 빠르게 찾기 어렵다.

## 3. Purpose and Value

사용자가 익숙한 단어로 검색해도 의미적으로 가까운 품목 정보를 즉시 확인하게 해, 검색 실패를 줄이고 배출 준비 시간을 단축한다. 현재 서비스가 제공 중인 유사 검색과 지역 필터 경험을 유지하되, 결과 신뢰도와 안내 일관성을 강화하는 것이 목적이다.

## 4. Goals

- 사용자가 1-2개의 일상 단어로도 관련 품목 후보를 찾을 수 있게 한다.
- 검색 결과에서 품목명, 분류, 규격, 지역, 수수료를 한 화면에서 비교 가능하게 한다.
- 시도/시군구 선택 기반 검색 흐름을 유지하면서, 지역 미선택 상태에서도 탐색이 가능하게 한다.
- 결과 없음/오류 상황에서 사용자가 다음 행동을 알 수 있도록 명확히 안내한다.

## 5. Non-Goals

- 결제, 신고, 수거 예약 등 행정 처리 기능 제공
- 지자체 원천 데이터의 생성/수정 업무 대체
- 민원 접수, 상담 챗봇, 폐기물 종류 교육 콘텐츠 제공

## 6. Target Users / Personas

- 1차 사용자: 가정에서 대형폐기물 배출을 준비하는 일반 시민
- 2차 사용자: 대형폐기물 배출 신고를 대신 처리하는 현장 관리자

## 7. Expected Behavior

### 7.1 Core User Journeys

1. 사용자가 검색어를 입력하고(선택적으로 지역을 고른 뒤) 유사 품목 목록을 확인해 배출 대상 품목과 수수료를 결정한다.
2. 사용자가 결과를 보고 같은 품목의 규격/지역 차이를 비교해 본인 상황에 맞는 항목을 선택한다.
3. 결과가 없거나 확신이 낮을 때, 사용자가 검색어를 바꿔 재검색하거나 지역 필터를 조정해 탐색을 이어간다.

### 7.2 Functional Expectations

- 사용자는 검색어 없이 검색을 시도할 수 없으며, 입력 필요 메시지를 즉시 확인할 수 있어야 한다.
- 서비스는 지역 목록(시도/시군구)을 제공하고, 시도 변경 시 하위 지역 선택 상태를 올바르게 갱신해야 한다.
- 검색 결과는 관련도 높은 순서로 상위 10개를 제공해야 하며, 각 항목에 품목명/분류/규격/지역/수수료를 포함해야 한다.
- 지역 미선택 상태에서도 전국 단위 검색 결과를 제공해야 한다.
- 서비스는 결과 없음 상태를 별도 화면 요소로 명확히 구분해 보여줘야 한다.
- 서비스 장애 또는 조회 실패 시, 사용자가 인지 가능한 오류 메시지를 제공해야 한다.
- 현재 제공 중인 상위 후보 중심 탐색 경험은 유지하되, 사용자가 결과 신뢰를 판단할 수 있는 일관된 표현(일치도 %)을 제공해야 한다.

### 7.3 Success Signals

- 사용자가 첫 검색 또는 1회 재검색 이내에 배출 대상 후보를 찾는 비율이 증가한다.
- 검색 후 이탈 없이 결과 목록을 확인하는 세션 비율이 증가한다.
- 결과 없음 또는 오류 후 재시도율이 유지/개선된다.
- 사용자 피드백에서 "원하는 품목을 찾기 쉽다"는 정성 의견이 증가한다.

## 8. Constraints

- 지자체별 기준 데이터 범위 내에서만 결과를 제공해야 한다.
- 공공정보 해석 오류를 줄이기 위해, 결과는 안내 목적이며 최종 기준은 관할 지자체 정책임을 전제로 해야 한다.
- 한국어 사용자 중심 서비스 톤과 용어 일관성을 유지해야 한다.

## 9. Assumptions

- 사용자 검색어는 짧은 단어 수준이며, 정확한 공식 품목명을 모르는 경우가 많다.
- 사용자는 최적 1건보다도 비교 가능한 상위 후보 목록을 선호한다.
- 지역 필터는 정확도를 높이지만, 필수 단계가 되면 탐색 진입 장벽이 커질 수 있다.
- 현재 서비스가 제공 중인 지역 필터, 결과 목록, 오류/빈 결과 메시지 구조는 유지 가치가 있다.
